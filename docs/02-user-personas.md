# User Personas

## Purpose

This document describes potential users of AdventureWorksAIWorkspace.

## Persona 1: Business Analyst

### Description

A user who wants to explore sales, product, customer, and regional data without manually writing SQL for every report.

### Needs

- Generate reports quickly.
- Ask follow-up questions.
- Compare business metrics.
- Export reports for meetings.
- Reuse previous reports.

### Pain Points

- Manual SQL writing is time-consuming.
- Dashboard tools require configuration.
- Data structures may not be obvious.

## Persona 2: Sales Manager

### Description

A user who needs sales insights, regional performance reports, and product performance analysis.

### Needs

- Understand sales trends.
- Compare product performance.
- Analyze regions and territories.
- Export reports to PDF.
- Reopen frequently used reports.

### Pain Points

- Does not want to write SQL.
- Needs quick answers during decision-making.
- Wants visual summaries, not raw rows.

## Persona 3: Product Manager

### Description

A user who wants to understand product sales, demand, and performance over time.

### Needs

- Analyze product categories.
- Compare product revenue.
- Identify top and underperforming products.
- Track historical trends.

## Persona 4: Admin User

### Description

A user responsible for managing access, monitoring generated SQL, and ensuring system safety.

### Needs

- Manage users.
- Review generated SQL history.
- Audit report generation.
- Ensure AdventureWorks is accessed safely.

## Persona 5: Developer / Demo User

### Description

A technical user evaluating the application as a portfolio project or demo of AI-assisted BI.

### Needs

- Understand architecture.
- Review AI workflow.
- Demonstrate functionality.
- Explain implementation decisions.
